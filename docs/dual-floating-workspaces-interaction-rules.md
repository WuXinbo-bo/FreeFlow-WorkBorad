# 双浮动工作区交互规则

本文用于约束 FreeFlow 桌面主界面中的两个主工作区交互逻辑：

- 左侧画布工作区
- 右侧对话工作区

两者保持在同一个 Electron 主窗口内运行，但在交互上视为两个同级、可漂浮、可独立缩放的主工作区，而不是传统的左右固定栏。

## 1. 基本原则

- 两个工作区允许自由拖拽、自由缩放、自由重叠。
- 两个工作区均为同级主工作区，不存在固定主副关系。
- 保持单窗口架构，不拆分为两个原生窗口。
- 不修改两个工作区内部具体内容，只调整外层交互与布局行为。

## 2. 层级规则

- 点击任一工作区，该工作区立即置顶。
- 拖拽、缩放、半屏、全屏、恢复时，目标工作区自动置顶。
- 当前激活工作区内部弹层优先高于另一工作区本体。

## 3. 拖拽规则

- 拖拽仅通过工作区顶部控制区触发。
- 拖拽过程中不触发画布平移、不触发对话区滚动。
- 使用软边界策略：
  - 允许工作区部分越出容器范围
  - 但必须始终保留最小可抓取区域
- 拖拽结束后持久化位置与层级。

## 4. 缩放规则

- 两个工作区均支持右下角缩放。
- 两个工作区均支持纵向边缘位移。
- 缩放最小尺寸固定，避免进入不可编辑状态。
- 缩放过程中实时同步相关依赖 UI 与窗口 shape。

## 5. 模式规则

每个工作区支持以下模式：

- `normal`
- `half-left`
- `half-right`
- `maximized`

说明：

- `normal` 为自由浮动状态。
- `half-left` / `half-right` 为半屏贴边工作区状态。
- `maximized` 为单工作区最大化显示状态。
- 每个工作区记录最近一次 `normal` 布局，用于恢复。

## 6. 恢复规则

- 工作区关闭后，可通过边缘恢复按钮重新打开。
- 工作区从半屏或全屏恢复时，优先恢复最近一次 `normal` 布局。
- 提供全局“恢复默认布局”入口。
- 默认布局恢复时，同时重置位置、尺寸、模式、可见性。

## 7. 兼容规则

- 保留既有 `left/right` 槽位数据结构，避免破坏现有内容绑定。
- 保留旧布局数据迁移能力。
- 新布局持久化以 `panelLayout v2` 为准。

## 8. 风险控制

- 保持软边界，不实现无限制越界漂浮。
- 面板几何变化必须持续同步窗口 shape。
- 任何模式切换后都必须保留可靠恢复路径。
