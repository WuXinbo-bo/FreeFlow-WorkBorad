# v1.0.0-rc PDF 导出阶段1基线说明

## 阶段目标

本阶段只做基线确认，结论是：

- 画布导出 PDF 可以做成独立封装
- 不需要修改现有画布 JSON 数据结构
- 不需要改动现有元素存储格式
- 可以复用现有的离屏渲染、图片预加载、图片导出保存链路
- 阶段1不进入代码实现，只确认接入点与禁止污染范围

---

## 一、现有可复用能力盘点

### 1. 导出范围与边界计算

- `public/src/engines/canvas2d-core/createCanvas2DEngine.js:3512`
  - `getExportBounds(items = [])`
  - 作用：计算导出元素的整体边界
  - 特点：已兼容普通元素与 `flowEdge`
- `public/src/engines/canvas2d-core/elements/index.js:211`
  - `getBoardBounds(items = [])`
  - 作用：计算整板元素边界
  - 结论：未来整板 PDF 导出可以优先复用这一层做 board 级范围计算，再视需要决定是否保留 `flowEdge` 的专项处理

### 2. 离屏 Canvas 渲染

- `public/src/engines/canvas2d-core/createCanvas2DEngine.js:3586`
  - `renderItemsToCanvas(items = [], options = {})`
  - 作用：将一组元素离屏渲染为 canvas
  - 当前已具备：
    - 自动计算导出边界
    - 自动加 padding
    - 创建离屏 canvas
    - 构造独立 view
    - 调用统一 renderer 输出
    - 支持背景填充与网格参数
  - 结论：这是未来 `renderBoardToCanvas()` 的最直接复用基础

### 3. 图片数据安全输出

- `public/src/engines/canvas2d-core/createCanvas2DEngine.js:3702`
  - `safeCanvasToDataUrl(canvas)`
  - 作用：将 canvas 转为 PNG dataUrl，并处理失败情况
- `public/src/engines/canvas2d-core/createCanvas2DEngine.js:3844`
  - `saveDataUrlAsImage(dataUrl, options = {})`
  - 作用：保存 PNG 到桌面端或 web 端下载
  - 结论：PDF 导出不应复用这个函数本体去“伪装保存 PDF”，但可以复用其保存策略判断逻辑

### 4. 图片资源预加载与补水

- `public/src/engines/canvas2d-core/createCanvas2DEngine.js:3740`
  - `preloadImagesForItems(items = [])`
  - 作用：导出前预加载图片资源
- `public/src/engines/canvas2d-core/createCanvas2DEngine.js:3761`
  - `hydrateImageItems(items = [])`
  - 作用：在导出前补足图片元素可渲染数据
  - 结论：PDF 导出必须复用这两层，否则含图片的画布导出结果不稳定

### 5. 现有图片导出主链路

- `public/src/engines/canvas2d-core/createCanvas2DEngine.js:3915`
  - `exportItemsAsImage(items = [], options = {})`
  - 当前顺序：
    1. `hydrateImageItems`
    2. `preloadImagesForItems`
    3. `renderItemsToCanvas`
    4. `safeCanvasToDataUrl`
    5. `saveDataUrlAsImage`
  - 结论：PDF 导出最适合沿用这条链路的前四步，再单独接 PDF 输出层

### 6. 画布截图链路

- `public/src/engines/canvas2d-core/createCanvas2DEngine.js:3980`
  - `startCanvasCapture()`
- `public/src/engines/canvas2d-core/createCanvas2DEngine.js:3991`
  - `runCanvasCapture(pointer)`
  - 结论：截图链路会回插图片进画布，属于交互型导出，不适合作为 PDF 导出主基础

## 二、现有保存链路与 UI 接入点

### 1. 保存 / 下载方式

#### 桌面端

- `public/src/engines/canvas2d-core/createCanvas2DEngine.js:3858`
  - `desktopShell.pickImageSavePath`
- `public/src/engines/canvas2d-core/createCanvas2DEngine.js:3864`
  - `desktopShell.writeFile`
- 结论：未来 PDF 导出应新增独立保存函数，沿用桌面端 `pick save path + writeFile` 模式

#### Web 端

- `public/src/engines/canvas2d-core/createCanvas2DEngine.js:3872`
  - `Blob + a.download`
- 结论：PDF 导出在 web 端可直接复用同类浏览器下载策略

### 2. 画布菜单入口

- `public/src/engines/canvas2d-core/ui/index.jsx:802`
  - `保存画布`
- `public/src/engines/canvas2d-core/ui/index.jsx:813`
  - `画布另存为`
- `public/src/engines/canvas2d-core/ui/index.jsx:860`
  - `exportMenuOpen`
- `public/src/engines/canvas2d-core/ui/index.jsx:868`
  - 当前“画布图片位置”子菜单
- 结论：未来“导出整张画布为 PDF”适合接在当前主菜单中，作为独立导出入口，不要混入“画布图片位置”设置菜单

### 3. 右键菜单导出入口

- `public/src/engines/canvas2d-core/createCanvas2DEngine.js:5548`
  - 右键菜单已有 `导出为图片`
- `public/src/engines/canvas2d-core/createCanvas2DEngine.js:5913`
  - 右键菜单动作分发到 `exportItemsAsImage`
- 结论：现有右键导出是“选区导出图片”，不是整板导出 PDF 的最佳入口；第一版 PDF 更适合先接顶部主菜单

### 4. 状态提示入口

- `public/src/engines/canvas2d-core/createCanvas2DEngine.js:1108`
  - `setStatus(text, tone = "neutral")`
- `public/src/engines/canvas2d-core/createCanvas2DEngine.js:1417`
  - `boardSaveToastAt / boardSaveToastMessage`
- `public/src/engines/canvas2d-core/ui/index.jsx:1107`
  - `canvas2d-save-toast`
- 结论：
  - PDF 导出过程提示，优先走 `setStatus`
  - 不建议复用 `boardSaveToast`，因为它是画布保存语义，不是导出语义

### 5. React Bridge 接入点

- `public/src/engines/canvas2d-core/reactBridge.js`
  - 当前 bridge 负责 UI 调度到 engine
  - 已有 `saveBoard`、`saveBoardAs`、`startCanvasCapture` 等接口
- 结论：未来阶段实现时，应新增独立桥接方法，例如：
  - `exportBoardAsPdf()`
  - 不应把 PDF 导出塞进 `saveBoard` 或 `startCanvasCapture`

## 三、阶段1明确禁止污染范围

本阶段以及后续 PDF 实现阶段，都不应直接改动以下高风险链路：

### 1. 画布数据结构

- `state.board`
- 画布 JSON 文件内容
- 元素结构：text / image / fileCard / flow / shape 等

禁止项：

- 不新增 PDF 导出字段到画布 JSON
- 不给元素写入导出状态
- 不把导出配置持久化进 board 数据

### 2. 自动保存链路

- `public/src/engines/canvas2d-core/createCanvas2DEngine.js:1455`
  - autosave timer
- `public/src/engines/canvas2d-core/createCanvas2DEngine.js:1513`
  - `saveBoard`

禁止项：

- 不允许 PDF 导出触发 `saveBoard`
- 不允许 PDF 导出改写 `boardDirty`
- 不允许 PDF 导出影响 autosave timer

### 3. 最近画布 / 当前画布路径链路

- `public/src/engines/canvas2d-core/createCanvas2DEngine.js:1325`
  - `persistBoardSelectionSetting`
- `public/src/engines/canvas2d-core/createCanvas2DEngine.js:1603`
  - `loadBoardFromPath`
- `public/src/engines/canvas2d-core/createCanvas2DEngine.js:1717`
  - `newBoard`
- `public/src/engines/canvas2d-core/createCanvas2DEngine.js:1769`
  - `openBoard/openBoardAtPath/ensureTutorialBoard`

禁止项：

- 不允许 PDF 导出修改当前画布路径
- 不允许 PDF 导出触发最近画布切换
- 不允许 PDF 导出介入教程画布判断

### 4. 教程系统

- `public/src/engines/canvas2d-core/tutorial-system/*`
- `public/src/engines/canvas2d-core/ui/index.jsx:173`
  - tutorial runtime 注入

禁止项：

- 不允许 PDF 导出写入教程状态
- 不允许 PDF 导出复用教程浮层状态
- 不允许把导出入口混进教程链路

### 5. 复制粘贴与拖拽导出链路

- `public/src/engines/canvas2d-core/createCanvas2DEngine.js:4198`
  - `copySelection`
- `public/src/engines/canvas2d-core/createCanvas2DEngine.js:4231`
  - `pasteInternalClipboard`
- `public/src/engines/canvas2d-core/createCanvas2DEngine.js:4389`
  - `buildExportDragPayload`
- `public/src/engines/canvas2d-core/createCanvas2DEngine.js:4437`
  - `startExportDragForSelection`

禁止项：

- 不允许 PDF 导出占用 clipboard
- 不允许 PDF 导出复用 native export drag 语义
- 不允许 PDF 导出向画布插回中间产物图片

### 6. 编辑态与视口态

禁止项：

- 不修改当前 `view`
- 不修改当前选区
- 不修改 hover / editing / draft / imageEditState
- 不重置缩放和视口
- 不触发 undo / redo 栈变化

## 四、阶段1接入结论

### 可以直接复用的基础

- 导出边界计算
- 离屏渲染
- 图片预加载
- 图片补水
- 桌面端写文件能力
- Web 端下载能力
- 统一状态提示能力

### 应单独新增的模块

- `public/src/engines/canvas2d-core/export/pdfExportOptions.js`
- `public/src/engines/canvas2d-core/export/renderBoardToCanvas.js`
- `public/src/engines/canvas2d-core/export/exportBoardAsPdf.js`

### 阶段1最终结论

该功能具备独立封装条件，建议后续实现严格遵守以下原则：

1. 只复用现有 renderer，不复用保存语义。
2. 只新增导出模块，不改画布数据结构。
3. 只加桥接入口和菜单入口，不碰最近画布、教程、复制粘贴、自动保存主链路。
4. PDF 导出状态只存在于运行时，不进入画布 JSON。

## 五、阶段2前置建议

进入下一阶段前，建议先固定 PDF 导出协议：

- `scope`
- `scale`
- `background`
- `includeGrid`
- `pageMode`
- `fileName`

并明确：

- 桌面端保存文件路径策略
- Web 端下载策略
- 超大画布降级规则
- 导出失败提示文案
