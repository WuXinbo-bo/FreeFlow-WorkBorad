# FreeFlow v1.0.0-rc 阶段 2：旧版兼容专项说明

## 1. 阶段目标

- 评估 `v0.8.0-beta` 用户升级到 `v1.0.0-rc` 时的数据连续性与启动连续性
- 区分“代码层兼容已具备”和“发布态兼容已完成验证”两类结论
- 形成后续发布前必须执行的兼容验证矩阵

## 2. 当前兼容结论

### 2.1 总体结论

- 当前代码已经具备从 `v0.8.0-beta` 向 `v1.0.0-rc` 过渡的兼容底座
- 用户数据目录、主设置、会话、AI 配置、权限、剪贴板队列、快捷键配置、旧画布元素结构，整体上已有延续能力
- 从代码阅读与现有升级脚本运行结果看，旧版用户直接进入新版的“基础兼容性”是成立的

### 2.2 当前结论边界

- 当前结论主要建立在：
  - 代码阅读
  - 内置升级脚本
  - 存储结构与画布读取链路复核
- 这还不等于：
  - 已完成正式安装包级别的全链路升级发布验证
- 因此当前阶段结论应表达为：
  - `当前代码兼容基础较稳，但正式发布前仍需跑完整兼容回归矩阵`

## 3. 已确认兼容基础

### 3.1 用户数据目录连续性

- 用户目录策略固定在：
  - `%USERPROFILE%\\FreeFlow`
- 关键数据路径固定在：
  - `%USERPROFILE%\\FreeFlow\\AppData`
  - `%USERPROFILE%\\FreeFlow\\CanvasBoards`
- 当前代码未改动这套目录归属规则，因此旧版用户数据可继续被新版读取

对应代码：
- [paths.js](D:\FreeFlow-WorkBoard\src\backend\config\paths.js)

### 3.2 应用身份连续性

- 当前应用标识保持稳定：
  - `productName = FreeFlow`
  - `appId = com.wuxinbo.freeflow`
- 这有利于桌面端保持同一应用身份和用户数据归属

对应代码：
- [package.json](D:\FreeFlow-WorkBoard\package.json)

### 3.3 启动阶段最近画布 / 教程画布兼容

- 当前启动链路已包含教程画布误写入“最近画布”的专项修正
- 若旧版把教程画布错误记录成最近画布，启动时会清除该路径并标记已展示教程
- 若最近画布路径不存在，启动时会自动回退到教程画布或默认画布逻辑

对应代码：
- [appStartupService.js](D:\FreeFlow-WorkBoard\src\backend\services\appStartupService.js)

### 3.4 后端存储版本化读取

- 以下关键存储均通过 `readVersionedJsonFile()` 做版本检查与 normalize：
  - UI 设置
  - 模型提供商设置
  - 模型画像
  - 权限设置
  - 会话存储
  - 剪贴板队列
- 这意味着旧版 JSON 在读入新版时会先被归一化，再按当前 schema 回写

对应代码：

### 3.5 真实画布文件结构兼容

- 实际运行中的画布读取链路是前端引擎，而不是后端 `canvasBoardService`
- 当前真实画布加载时会：
  - 读取本地 JSON
  - `JSON.parse`
  - 进入 `normalizeBoard(parsed)`
- 元素兼容层已包含旧字段映射：
  - `kind -> type`
  - `file -> fileCard`
  - `mind -> mindNode`
  - `node -> flowNode`
  - `edge -> flowEdge`
  - `richtext / rich -> text`

对应代码：
- [createCanvas2DEngine.js](D:\FreeFlow-WorkBoard\public\src\engines\canvas2d-core\createCanvas2DEngine.js)
- [elements/index.js](D:\FreeFlow-WorkBoard\public\src\engines\canvas2d-core\elements\index.js)

## 5. 当前风险与限制

### 5.1 版本元信息未统一

- 当前版本日志已切换到 `v1.0.0-rc`
- 但源码版本仍是：
  - `package.json.version = 0.8.0-beta`
- 当前 `release/` 里也仍保留旧版包名

### 5.2 教程本地状态不保证平滑延续

- 教程状态存储在浏览器 `localStorage`
- 当前教程版本仍是：
  - `TUTORIAL_VERSION = 0.8.1`
- 若版本号变化，教程状态会按“版本不一致”逻辑清理或重置部分字段

对应代码：
- [tutorialTypes.js](D:\FreeFlow-WorkBoard\public\src\tutorial-core\tutorialTypes.js)
- [tutorialStateStore.js](D:\FreeFlow-WorkBoard\public\src\tutorial-core\tutorialStateStore.js)

### 5.3 画布复制粘贴链路属于发布前重点风险项

- 近期围绕：
  - 画布内部结构化复制
  - 外部系统剪贴板复制
  - 保留格式粘贴
  - 纯文本退化
- 已做过多轮修正
- 该链路当前不影响“旧版用户数据能否读出来”，但会影响发布质量，因此必须列入高优先级专项回归
