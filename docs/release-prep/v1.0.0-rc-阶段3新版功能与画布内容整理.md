# FreeFlow v1.0.0-rc 阶段 3：新版功能与画布内容整理

## 1. 阶段目标

- 明确 `v1.0.0-rc` 必须进入发布范围的功能
- 梳理画布升级主线，区分“必须交付”和“可延期项”
- 为后续 `README.md`、发布说明、打包验收和版本日志收口提供统一范围依据

## 2. 本阶段总原则

- `v1.0.0-rc` 不是继续零散修补，而是一次“结构收敛 + 核心体验补完”的版本
- 新版范围优先保证：
  - 启动稳定
  - 画布主链路稳定
  - 桌面工作台结构稳定
  - 教程系统可用
  - AI 镜像主交互稳定
- 不追求把所有潜在功能都塞进 `v1.0.0-rc`
- 超出正式发布主线的增强项，应明确后移到 `v1.0.1`

## 3. v1.0.0-rc 必进功能范围

### 3.1 桌面工作台主结构

- 左右主面板自由布局
  - 拖拽移动
  - 独立缩放
  - 上下位移
  - 左右换位
- 全局浮层统一 overlay 层
  - 设置抽屉
  - 更多菜单
  - 映射控制
  - 教程中心
  - 穿透提示
- 桌面窗口 shape 主线收敛
  - 浮层显示与点击区域同步
  - 透明区域穿透正确
  - 全屏 / 恢复 / 重排后 shape 跟随稳定

### 3.2 画布核心体验

- 最近画布恢复逻辑
- 教程画布首次引导逻辑
- 画布内多选复制粘贴保留格式
- 画布内复制与程序外复制的优先级切换
- 画布内全局搜索
- 富文本编辑能力增强
  - 工具条稳定布局
  - 大字号支持
  - 链接 / 样式 / 输入态尺寸跟随
- 画布基础交互一致性
  - 空白区右键菜单
  - 右键拖动画布
  - 中键拖动画布
  - 视角缩放

### 3.3 教程系统

- 教程中心主入口可用
- 教程中心分区结构稳定
  - 主界面教程
  - 画布教程
  - AI 镜像教程
- 快捷键速查文档入口接入教程中心
- `v1.0.0-rc` 教程画布基线冻结
  - 当前使用：
    - `C:\Users\lenovo\FreeFlow\CanvasBoards\FreeFlow教程画布.json`

### 3.4 AI 镜像工作区

- 顶部切换、映射控制、更多菜单、设置抽屉在布局变化下稳定可用
- 嵌入目标切换、暂停 / 恢复、关闭 / 重建链路可用
- 右侧面板拉伸、关闭、换位后不出现明显错位和不可点击状态

### 3.5 旧版本兼容基础

- 保持旧版用户数据目录延续
- 保持旧版关键设置可读取
- 保持旧版画布基本结构可读取
- 保持旧版会话与 AI 配置可读取

## 4. 画布升级主线

### 4.1 画布主线目标

- 让画布在 `v1.0.0-rc` 达到“可稳定日用”的正式版水位
- 当前画布升级不再只追求新功能数量，而是优先解决：
  - 打开链路
  - 编辑链路
  - 复制粘贴链路
  - 搜索链路
  - 教程引导链路

### 4.2 画布必交付能力

- 画布启动：
  - 最近画布优先
  - 教程画布只在首次引导使用
- 画布编辑：
  - 普通文本
  - flow 节点文本
  - 富文本工具条
- 画布复制粘贴：
  - 结构化复制
  - 外部复制切换
  - 排版保留
- 画布搜索：
  - 关键词搜索
  - 内容定位
  - 搜索入口可见
- 画布教程：
  - 教程中心入口
  - 快捷键速查
  - `v1.0.0-rc` 教程画布基线

### 4.3 画布当前发布风险项

- 多选复制后保留格式粘贴链路刚经历多轮修复，仍是发布前最高优先级回归项
- 画布内复制与系统剪贴板切换逻辑必须进行真实场景验证
- 富文本大字号与重开后的显示一致性需专项回归
- 教程画布内容更新后，需要确认首次引导和最近画布逻辑不会互相覆盖

## 5. 可延期到 v1.0.1 的内容

### 5.1 可继续增强但不阻塞 rc 发布的内容

- 画布教程内容继续扩充更多章节
- AI 镜像工作区更细颗粒度的交互优化
- 更复杂的富文本高级能力
  - 更丰富的块级样式
  - 更高级的排版规则
- 非核心视觉风格继续打磨
- 更广覆盖的自动化测试体系

### 5.2 明确不在 v1.0.0-rc 强行完成的事项

- 所有边缘场景下的交互微调一次性全部做完
- 把所有历史遗留问题在一个版本内全部清零
- 过度扩张新的业务模块或新工作区

## 6. 当前版本范围结论

- `v1.0.0-rc` 的核心定位是：
  - 桌面工作台结构稳定
  - 画布主链路稳定
  - 教程系统可用
  - AI 镜像工作区可稳定使用
  - 旧版用户可平滑过渡
- 这次版本的重点不是继续堆功能数量，而是把 FreeFlow 的主干体验第一次收口到可正式发布的状态
