# FreeFlow v1.0.0-rc 阶段 5：发布验收与发包执行方案

## 1. 阶段目标

- 在不再扩散需求的前提下，对 `v1.0.0-rc` 做正式打包前验收
- 完成旧版升级验证、核心链路回归、安装包产出、发布说明收口
- 形成一套可以重复执行的发包顺序

## 2. 阶段 5 执行顺序

### 第一步：回归前冻结

- 暂停继续插入非必要新功能
- 仅允许修复阻塞发布的问题
- 固定当前教程画布基线：
  - `C:\Users\lenovo\FreeFlow\CanvasBoards\FreeFlow教程画布.json`

### 第二步：执行回归验收

- 按正式验收清单逐项验证：
  - 启动链路
  - 最近画布 / 教程画布
  - 复制粘贴
  - 画布搜索
  - 富文本编辑
  - 教程中心
  - AI 镜像
  - 主面板布局
  - 升级兼容

### 第三步：正式构建

- 执行：
  - `npm run dist:win`
- 期望产物：
  - `FreeFlow-v1.0.0-rc-x64.exe`
  - `FreeFlow-v1.0.0-rc-x64-portable.exe`

### 第四步：安装包验证

- 用旧版安装包做覆盖安装
- 验证新版首次启动与二次启动
- 验证卸载不会删除用户业务数据

### 第五步：发布资料收口

- 固化版本日志
- 固化发布说明
- 固化升级说明
- 固化 README 与 LICENSE

## 3. 阶段 5 必验风险项

- 最近画布打开链路是否稳定
- 教程画布是否只在首次引导打开
- 多选复制粘贴是否保留排版
- 程序外复制后是否还能正常覆盖系统剪贴板
- 画布全局搜索是否可定位正确元素
- 富文本超大字号保存后重开是否一致
- 主面板拖拽、缩放、换位后状态是否正常保留
- AI 镜像相关顶部控件是否仍可点击
