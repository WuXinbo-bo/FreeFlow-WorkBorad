FreeFlow 打包资源目录。

当前发布目标版本：`v1.0.8-rc`

Windows 应用图标已由打包脚本自动生成到这里：

- `icon.ico`
- `icon.png`

当前正式产品名为 `FreeFlow`，产品描述为“`一站式打造属于你的专属办公白板`”。

当前 Windows 安装策略已固定为：

- 产品名固定为 `FreeFlow`
- `appId` 固定为 `com.wuxinbo.freeflow`
- 安装目录固定为当前用户：`%LOCALAPPDATA%\Programs\FreeFlow`
- 重新运行新版安装包时，默认覆盖同一安装目录
- 卸载仅移除程序文件，不删除 `C:\Users\<用户名>\FreeFlow\...` 下的用户业务数据

因此当前 `v1.0.8-rc` 正式发布后的升级方式是：

- 用户下载新的 `FreeFlow-v1.0.8-rc-x64.exe`
- 直接运行安装覆盖旧版本
- 打开后继续使用原有画布与配置

当前打包产物命名规则：

- `FreeFlow-v1.0.8-rc-x64.exe`
- `FreeFlow-v1.0.8-rc-x64-portable.exe`

发版前固定验证流程：

1. 自动验证
   - 运行 `npm run check:desktop-upgrade`
   - 该脚本会模拟旧版用户数据环境，并检查：
   - 最近画布是否保留
   - 教程画布回退是否正常
   - 主题设置是否保留
   - 快捷键是否保留
   - AI 配置是否保留
   - 历史会话是否保留
   - 老画布是否能正常读取

2. 覆盖安装手工实测
   - 安装旧版 `FreeFlow`
   - 准备一份真实用户数据并启动旧版
   - 关闭旧版
   - 运行新版安装包覆盖安装
   - 启动新版后检查：
   - 最近画布是否仍被打开
   - 教程画布回退是否正常
   - 主题设置是否保留
   - 快捷键是否保留
   - AI 配置是否保留
   - 历史会话是否保留
   - 老画布是否能正常打开

当前 `pack:win` / `dist:portable` / `dist:win` 都已接入自动升级验证脚本。
