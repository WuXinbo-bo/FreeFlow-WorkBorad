# FreeFlow

</p>

FreeFlow 是一个以桌面端为核心的可视化工作白板，整合了自研画布引擎、AI 助手工作区、AI 镜像工作区，以及可自由布局的桌面工作台结构，目标是把思考、记录、整理、协作与 AI 操作集中到一个统一环境中。

当前发布目标版本：`v1.0.0-rc`

FreeFlow 的核心特色之一，是支持把文本、图片、文件等各种素材直接拖入画布进行整理与协作，同时支持给图片和文件补充备忘录内容，把零散素材快速沉淀成可视化工作记录。除此之外，画布内还提供全局搜索能力，可快速检索文本、节点、文件名与备注内容；配合 AI 镜像嵌入集成能力，可以把外部 AI 工作区直接纳入同一桌面工作流中使用。

## 项目简介

FreeFlow 面向这样的使用场景：

- 在自由画布中做视觉化记录、结构化整理和发散思考
- 在同一工作区中调用 AI 助手进行写作、分析和任务执行
- 将外部 AI 页面或目标窗口映射到应用内部进行操作
- 保留个人桌面工作布局、画布、会话与设置，形成长期工作环境

它不是单纯的聊天工具，也不是传统意义上的简单白板。  
FreeFlow 的核心方向是一个融合以下能力的桌面工作系统：

- 自研画布引擎
- 桌面 AI 助手壳层
- AI 镜像工作区
- 教程驱动的引导系统

## 核心功能

- 自研画布引擎，支持文本、节点、图片、文件卡片、流程连线和图形元素
- 支持文本、图片、文件等各种素材直接拖入画布，快速整理、标注和协作
- 支持给图片与文件补充备忘录内容，适合做随手记录、资料备注和灵感收集
- 画布内全局搜索，可快速查找文本、节点、文件名、标签和备注内容
- 富文本编辑能力增强，支持大字号和更稳定的编辑工具条布局
- 画布内多选结构化复制粘贴
- 启动时优先恢复最近画布，同时将教程画布限定为首次引导用途
- 教程中心，区分主界面教程、画布教程和 AI 镜像教程
- 快捷键速查入口，直接查看常用快捷键，无需进入完整教程流程
- 自由布局桌面工作台，左右主面板支持拖拽、缩放、换位
- 全局浮层架构，统一管理菜单、抽屉、教程中心、映射控制和穿透提示
- AI 镜像工作区，支持嵌入和操作外部 AI 目标，把 AI 能力直接集成到桌面工作台
- 桌面透明窗口 shape 同步能力，提升点击穿透与浮层交互稳定性

## v1.0.0-rc 版本重点

- 教程中心结构重整，并增加快捷键速查入口
- AI 镜像顶部控制区、更多菜单与设置链路稳定性提升

## 项目结构

- `electron/`
  - Electron 主进程、桌面桥接、AI 镜像与嵌入逻辑
- `public/src/engines/canvas2d-core/`
  - 自研画布引擎、渲染器、编辑逻辑、搜索、教程入口
- `public/src/runtime/`
  - 主界面运行时、主面板布局、窗口 shape 同步、教程宿主
- `src/backend/`
  - 持久化服务、用户数据存储、启动迁移逻辑、运行时配置
- `data/`
  - 教程画布等预置资源
- `docs/`
  - 版本日志、发布准备文档、规划说明

## 本地启动

安装依赖：

```powershell
npm install
```

启动桌面版：

```powershell
npm run start:desktop
```

如需前端开发调试，可单独启动本地服务：

```powershell
npm start
```

该模式主要用于开发调试，不作为正式使用入口。FreeFlow 的正式使用场景是桌面端应用，而不是浏览器模式。

## 打包方式

构建画布 UI：

```powershell
npm run build:canvas2d-ui
```

打包 Windows 安装版：

```powershell
npm run dist:win
```

当前打包产物命名已统一为：

- `FreeFlow-v1.0.0-rc-x64.exe`
- `FreeFlow-v1.0.0-rc-x64-portable.exe`

## 用户数据兼容

FreeFlow 当前桌面端用户数据目录为：

```text
%USERPROFILE%\FreeFlow
```

关键路径包括：

- `%USERPROFILE%\FreeFlow\AppData`
- `%USERPROFILE%\FreeFlow\CanvasBoards`

当前项目的 `v1.0.0-rc` 兼容目标是尽量保证 `v0.8.0-beta` 用户在进入新版时仍可继续使用原有：

- 设置
- 会话
- 模型配置
- 权限配置
- 画布文件

最终发布质量仍以正式发布前完整回归测试为准。

## 环境变量说明

部分模型相关能力可通过 `.env` 或环境变量配置。

例如使用 Ollama：

```powershell
$env:OLLAMA_BASE_URL="http://127.0.0.1:11434"
$env:OLLAMA_MODEL="你的本地模型名"
npm start
```

例如使用 LM Studio：

```powershell
$env:AI_PROVIDER="lmstudio"
$env:OPENAI_COMPAT_BASE_URL="http://127.0.0.1:1234/v1"
$env:AI_MODEL="你的本地模型名"
npm start
```

例如使用 BigModel / 智谱：

```powershell
AI_PROVIDER=bigmodel
BIGMODEL_BASE_URL=https://open.bigmodel.cn/api/paas/v4
BIGMODEL_API_KEY=你的 API Key
BIGMODEL_MODEL=glm-4.7-flash
AI_MODEL=glm-4.7-flash
```
